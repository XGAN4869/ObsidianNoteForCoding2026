## 核心概念：pages vs components

| 类型 | 说明 |
|------|------|
| `pages/` 下的 .vue | 有独立 URL，可通过 `uni.navigateTo` 跳转，有**页面生命周期** |
| `components/` 下的 .vue | 没有 URL，只是作为标签引用，只有**组件生命周期** |

- `pages/` = 能用 `router.push('/xxx')` 跳转到的页面
- `components/` = `import` 进来当标签用的东西

---

## 核心结论：uni-app 是"伪多页面"，Vue 是"真单页面"

### Vue 是单实例，SPA 架构

```
Vue 项目
┌────────────────────────────────────┐
│   index.html（唯一入口，永远不变）  │
│   └── Vue App 实例（整个项目一个）  │
│        └── Vue Router              │
│             └── Home 组件          │
│             └── Detail 组件（切换） │
└────────────────────────────────────┘
```

你刷新页面，Vue 实例重新创建。你点链接跳转，Vue Router 只是**切换组件**，Vue 实例本身和 index.html **永远不变**。

### uni-app 是多实例，每个页面都是独立创建

```
uni-app 小程序
┌─────────────────┐  ┌─────────────────┐
│  小程序进程      │  │  Webview 1       │
│  （一直在运行）  │──│  pages/index    │
│                 │  │  （页面实例A）   │
│                 │  └─────────────────┘
│                 │  ┌─────────────────┐
│                 │  │  Webview 2       │
│                 │  │  pages/detail   │
│                 │  │  （页面实例B）   │ ← navigateTo 新建
│                 │  └─────────────────┘
└─────────────────┘
```

- `navigateTo` = **创建全新的页面实例**（新的 webview）
- `navigateBack` = **销毁当前页面实例**（当前 webview 没了）
- 页面实例是**真实独立的**，有自己独立的 JS 上下文和状态

### Vue Router vs uni-app 页面栈

```
Vue Router 导航：
index.html + Vue App 实例 ─┬─ Home 组件 ─┬─ About 组件
                          │  (切换)      │  (替换)
                          └─────────────┘
              Vue 实例始终存在，只是组件在切换

uni-app 页面导航：
index.html + 小程序进程 ─┬─ [index 页面实例] ─┬─ [detail 页面实例] ← navigateTo 新建
                         │  onLoad(1次)      │  onLoad(1次)
                         │  onShow          │  onShow
                         │  ...             │  ...
                         │                  │  onUnload ← navigateBack 销毁
                         └──────────────────┘
              每个页面都是独立实例，不只是组件切换
```

## 页面生命周期 vs Vue 路由守卫

uni-app 的页面生命周期，**对标的是 Vue 路由导航**，不是 Vue 组件自身生命周期：

| uni-app 页面生命周期 | 对应的 Vue 概念 | 说明 |
|---------------------|----------------|------|
| `onLoad` | `beforeRouteEnter` + `onMounted` 组合 | 页面进入时触发，只执行一次，可接参数 |
| `onShow` | **无等价** | uni-app 多端特有，页面每次可见都触发 |
| `onReady` | `nextTick()` + `onMounted` | DOM 渲染完成 |
| `onHide` | `beforeRouteLeave` | 页面离开/隐藏 |
| `onUnload` | `onUnmounted` + 页面销毁 | 页面卸载 |

### 为什么 uni-app 需要单独的页面生命周期？

因为 uni-app 的页面是**真正独立创建的 webview 实例**，不是 Vue 那种组件替换。

| 行为 | uni-app | Vue |
|------|---------|-----|
| 打开新页面 | 创建全新页面实例，onLoad 执行 | 组件挂载，onMounted 执行 |
| 页面切到后台 | onHide 触发（实例还在） | 无这个概念（SPA 不存在"切后台"） |
| 切回前台 | onShow 触发（实例还在） | 无这个概念 |
| 关闭页面 | onUnload 销毁实例 | onUnmounted 销毁组件（但 Vue 实例还在） |

**核心差异：Vue 是单实例，uni-app 是多实例。**

## onLoad 和 Vue 的 getData 的关系

**uni-app 的 `onLoad` ≈ Vue 的 `beforeRouteEnter` + `onMounted` + `getData()` 组合。**

```javascript
// Vue 后台管理 - 刚进页面调用 getData 拿数据
onMounted(() => {
  getData()  // 相当于 Vue 的 getData()
})
```

```javascript
// uni-app - onLoad = 页面级别的 onMounted
onLoad(() => {
  loadData()  // 相当于 Vue 的 getData()
})
```

**本质上都是：页面进入 → 拿数据 → 渲染页面。**

区别：
- Vue 后台：刷新整个页面，不存在"切后台回来再刷新"的需求
- uni-app：切后台再回来，`onShow` 会再次触发

> **uni-app 的 `onLoad` ≈ Vue 的 `getData()`，`onShow` 是 uni-app 独有的"页面重新可见"时再处理一次的能力。**

## 三层生命周期总览

```
应用生命周期（App.vue）  →  onLaunch → onShow → onHide
        ↓
页面生命周期（pages/）    →  onLoad → onShow → onReady → onHide → onUnload
        ↓
组件生命周期（任意.vue）  →  created → mounted → updated → unmounted
```

---

## 应用生命周期（App.vue）

```javascript
export default {
  onLaunch() { console.log('App 启动') },
  onShow()   { console.log('App 显示') },
  onHide()   { console.log('App 隐藏') }
}
```

## 页面生命周期（仅 pages/ 下有效）

| 生命周期 | 触发时机 | 能否接参数 | 典型用法 |
|----------|----------|------------|----------|
| `onLoad` | 页面加载完成 | ✅ `options` | 请求初始数据、接收跳转参数 |
| `onShow` | 页面显示（每次前台切换都触发） | ❌ | 刷新数据、曝光统计 |
| `onReady` | 页面初次渲染完成 | ❌ | 获取 DOM（`createSelectorQuery`） |
| `onHide` | 页面隐藏（切后台/跳转） | ❌ | 暂停定时器、保存状态 |
| `onUnload` | 页面卸载（关闭/redirectTo） | ❌ | 清理资源 |
| `onReachBottom` | 页面触底 | ❌ | 加载更多 |
| `onPageScroll` | 页面滚动 | ❌ | 显示/隐藏回到顶部按钮 |

**执行顺序：**
```
进入：onLoad(1次) → onShow(可多次) → onReady(1次)
离开：onHide / onUnload
```

### 项目实战：生命周期 + API 组合

**1. like.vue - onLoad + showLoading + API 请求**

```javascript
// 喜欢页 - 加载数据
onLoad(() => {
  loadLikeList()
})

const loadLikeList = () => {
  uni.showLoading({ title: '加载中...' })
  likeList().then(res => {
    uni.hideLoading()
    if (res && res.code === 1) linkList.value = res.data
  }).catch(() => {
    uni.hideLoading()
    uni.showToast({ title: '加载失败', icon: 'none' })
  })
}
```

**2. detail.vue - onLoad 接收编码参数 + 路由跳转**

```javascript
// 详情页 - 接收并解析编码的跳转参数
onLoad((opt) => {
  if (opt && opt.item) {
    const data = JSON.parse(decodeURIComponent(opt.item))
    Object.assign(itemData, data)
  }
})

// 详情页 - 跳转路线页（navigateTo + 传参）
const goLine = (item) => {
  uni.navigateTo({
    url: `/pages/line/line?id=${item.id}`
  })
}
```

**3. line.vue - onLoad + onHide 组合**

```javascript
// 路线页 - 加载时请求数据
onLoad((props) => {
  if (props.id) fetchProjectDetail(props.id)
})

// 路线页 - 离开时暂停操作
onHide(() => {
  console.log('离开路线页，暂停地图更新')
})

// 返回上一页
const goBack = () => {
  uni.navigateBack()
}
```

**4. index.vue - onPageScroll + onReachBottom**

```javascript
// 首页 - 滚动显示回到顶部按钮
onPageScroll((e) => {
  showTopBtn.value = e.scrollTop > 400
})

// 首页 - 触底加载更多
onReachBottom(() => {
  uni.showLoading({ title: '加载中...' })
  setTimeout(() => {
    addRandomData()
    uni.hideLoading()
  }, 800)
})
```

**5. App.vue - 应用级生命周期**

```javascript
// App.vue - 应用启动/显示/隐藏
export default {
  onLaunch() {
    console.log('App 启动')  // 首次启动
  },
  onShow() {
    console.log('App 显示')  // 从后台切回前台
  },
  onHide() {
    console.log('App 隐藏')  // 切到后台
  }
}
```

## 组件生命周期（Vue 3 完全相同）

| 生命周期 | 说明 |
|----------|------|
| `onMounted` | DOM 挂载完成 |
| `onUpdated` | 数据更新后触发 |
| `onUnmounted` | 组件卸载销毁 |

```javascript
<script setup>
  import { onMounted, onUpdated, onUnmounted } from 'vue'
  onMounted(() => { console.log('DOM 挂载完成') })
  onUpdated(() => { console.log('数据更新了') })
  onUnmounted(() => { console.log('组件销毁') })
</script>
```

## Vue 3 等价概念对比

uni-app 页面生命周期是**多端小程序环境**的需求产物，Vue 3 作为 SPA 框架没有完全对应概念。

| uni-app | Vue 3 | 能否替代 |
|---------|-------|----------|
| `onLoad` | `beforeRouteEnter` + `onMounted` | ⚠️ 部分等价，`onLoad` 直接接参数 |
| `onShow` | **无等价** | ❌ SPA 不存在切后台概念 |
| `onReady` | `nextTick()` + `onMounted` | ⚠️ 接近 |
| `onHide` | **无等价** | ❌ SPA 没有"页面隐藏"概念 |
| `onUnload` | `onUnmounted` | ✅ 最近似 |
| `onReachBottom` | **无等价** | ❌ 需手写滚动监听 |
| `onPageScroll` | **无等价** | ❌ 需手写 scroll 事件 |

### 为什么需要 `onShow`？

Vue 3 是 SPA，切换路由 = 销毁旧组件 + 创建新组件。但 uni-app 是多端应用，用户可能切到其他 App、点击 tabBar 切换、锁屏解锁——这些都不走完整导航，所以需要 `onShow` 监听每次页面显示。

### Vue 3 等价示例

```javascript
// onReady ≈ nextTick
onMounted(() => { import('vue').then(({ nextTick }) => nextTick(() => { /* DOM ready */ })) })

// onReachBottom - 手动监听滚动
const handleScroll = () => {
  if (window.scrollY + window.innerHeight >= document.documentElement.scrollHeight - 100) loadMore()
}
onMounted(() => window.addEventListener('scroll', handleScroll))
onUnmounted(() => window.removeEventListener('scroll', handleScroll))
```

## 总结对比

| 特性 | 页面生命周期 | 组件生命周期 |
|------|-------------|-------------|
| 生效位置 | 仅 `pages/` 下 | 任意 `.vue` |
| 能接跳转参数 | ✅ `onLoad(options)` | ❌ |
| `pages/` 下页面 | ✅ 页面 + 组件生命周期都能用 | ✅ 页面 + 组件生命周期都能用 |
| `components/` 下组件 | ❌ | ✅ 只有组件生命周期 |

| 对应关系 | uni-app | Vue 3 |
|---------|---------|-------|
| 页面进入拿数据 | `onLoad` | `beforeRouteEnter` + `onMounted` + `getData()` |
| 页面重新可见 | `onShow` | **无等价** |
| 页面离开 | `onHide` / `onUnload` | `beforeRouteLeave` / `onUnmounted` |

> **uni-app = Vue 语法 + 真实多页面实例 + 多端原生能力**。Vue 的 SPA 架构下整个项目只有一个实例，而 uni-app 每个页面都是独立实例，所以需要 `onLoad/onShow/onHide/onUnload` 这套页面级生命周期来管理每个页面的创建、显示、隐藏、销毁。
