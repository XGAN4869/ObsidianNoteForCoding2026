# 路由跳转

## 官方文档

- API 跳转：https://uniapp.dcloud.net.cn/api/router.html#navigateto
- 组件跳转：https://uniapp.dcloud.net.cn/component/navigator.html

---

## 跳转方式总览

uni-app 路由跳转有两种方式：

| 方式 | 示例 | 说明 |
|------|------|------|
| **API 方式** | `uni.navigateTo({ url: '/pages/detail/detail' })` | JS 代码调用 |
| **组件方式** | `<navigator url="/pages/detail/detail">跳转</navigator>` | 声明式，和 Vue 的 `<router-link>` 一样 |

> **本质一样**，组件方式最终也是调用对应的 API。

---

## API 跳转方式

| API | 说明 | 页面栈变化 |
|-----|------|-----------|
| `uni.navigateTo()` | 保留当前页面，跳转到应用内某个页面 | **压栈** |
| `uni.redirectTo()` | 关闭当前页面，跳转到应用内某个页面 | **替换栈顶** |
| `uni.reLaunch()` | 关闭所有页面，打开到应用内某个页面 | **清栈后压入** |
| `uni.switchTab()` | 跳转到 tabBar 页面，关闭其他所有非 tabBar 页面 | **切 Tab** |
| `uni.navigateBack()` | 关闭当前页面，返回上一页面或多级页面 | **弹栈** |

### 语法示例

```javascript
// 跳转到非 tabBar 页面（保留当前页，页面栈 +1）
uni.navigateTo({ url: '/pages/detail/detail?id=1' })

// 跳转到非 tabBar 页面（关闭当前页，页面栈 -1）
uni.redirectTo({ url: '/pages/detail/detail?id=1' })

// 跳转到 tabBar 页面（必须用 switchTab，否则 tabBar 不生效）
uni.switchTab({ url: '/pages/my/my' })

// 关闭所有页面，重新打开页面（慎用，页面栈清空）
uni.reLaunch({ url: '/pages/index/index' })

// 返回上一页（页面栈 -1）
uni.navigateBack()

// 返回多级页面（页面栈 -n）
uni.navigateBack({ delta: 2 })
```

## 组件跳转方式（navigator）

`navigator` 是 uni-app 内置的导航组件，和 API 方式一一对应。

| open-type | 对应 API | 说明 |
|-----------|----------|------|
| `navigate`（默认） | `uni.navigateTo()` | 保留当前页，跳转到新页 |
| `redirect` | `uni.redirectTo()` | 关闭当前页，跳转到新页 |
| `switchTab` | `uni.switchTab()` | 跳转到 tabBar 页面 |
| `reLaunch` | `uni.reLaunch()` | 关闭所有页面，重新打开 |
| `navigateBack` | `uni.navigateBack()` | 返回上一页 |
| `exit` | — | 退出小程序（很少用） |

### 语法示例

```vue
<!-- 跳转到非 tabBar 页面，默认 open-type="navigate" -->
<navigator url="/pages/detail/detail?id=1">跳转详情</navigator>

<!-- 跳转到 tabBar 页面，必须 open-type="switchTab" -->
<navigator url="/pages/my/my" open-type="switchTab">我的</navigator>

<!-- 关闭当前页并跳转 -->
<navigator url="/pages/detail/detail" open-type="redirect">关闭当前页跳转</navigator>

<!-- 返回上一页 -->
<navigator open-type="navigateBack">返回</navigator>

<!-- 带参数跳转 -->
<navigator url="/pages/detail/detail?item={{encodedItem}}" hover-class="none">
  点击跳转
</navigator>
```

### 注意事项

- `hover-class="none"` 可去除点击时的默认样式
- `target` 属性可以指定在小程序还是 App 中生效（默认本系统）
- 组件方式适合在模板中简单跳转，复杂逻辑还是用 API

## 与 Vue 3 对比

### 页面栈说明

uni-app 的页面栈是**真实的栈结构**：

```
navigateTo    →  push   →  [home, detail, detail]     压栈
redirectTo    →  replace→  [home, detail]            替换栈顶
reLaunch      →  全新栈 →  [index]                    清栈后压入
navigateBack  →  pop   →  [home]                      弹栈
switchTab     →  切Tab →  关闭所有非 Tab 页，切换 Tab
```

> ⚠️ 小程序页面栈限制**最多 10 层**，超过会崩溃。

## 与 Vue 3 对比

| Vue 3 | uni-app | 说明 |
|-------|---------|------|
| `router.push()` | `uni.navigateTo()` | 压栈跳转 |
| `router.replace()` | `uni.redirectTo()` | 替换当前页 |
| `router.replaceAll()` | `uni.reLaunch()` | 清栈后全新打开 |
| `router.push('/tab')` | `uni.switchTab()` | Tab 切换（uni-app 特有） |
| `router.go(-1)` | `uni.navigateBack()` | 返回 |
| `router.go(-n)` | `uni.navigateBack({ delta: n })` | 返回 n 层 |
| `<router-link>` | `<navigator>` | 组件式跳转 |

### 关键区别

**1. TabBar 页面必须用 `switchTab`**

```javascript
// Vue 3：tabBar 页面和其他页面一样用 push
router.push('/tab')  // ✅ 正常

// uni-app：必须用 switchTab，否则 tabBar 不显示
uni.switchTab({ url: '/pages/my/my' })  // ✅ 正确
uni.navigateTo({ url: '/pages/my/my' })  // ❌ tabBar 不生效
```

**2. 页面栈是真实的，Vue Router 是虚拟的**

```javascript
// Vue 3：可以随时 push，返回靠 go(-1)
// 实际上只是操作浏览器的 history API，没有真实页面限制

// uni-app：页面栈是真实的 OS 资源
// navigateTo 超过 10 次，小程序会崩溃
```

**3. 参数传递方式不同**

```javascript
// Vue 3：params / query（自动处理编码）
router.push({ path: '/detail', query: { id: 1 } })
// 获取：route.query.id

// uni-app：直接拼在 URL 里，需手动编解码
uni.navigateTo({ url: '/pages/detail/detail?id=1' })
// 获取：onLoad((options) => { options.id })
```

**为什么要 JSON？**

URL 参数只能是**字符串**，不能直接传递对象。

```javascript
// 假设我们有一个对象
const item = { name: "景点'name", price: 100 }

// ❌ 错误写法：对象不能直接作为 URL 参数
uni.navigateTo({
  url: `/pages/detail/detail?item=${item}`  // item 会变成 [object Object]
})

// ✅ 正确写法：先把对象转成 JSON 字符串，再拼到 URL 里
uni.navigateTo({
  url: `/pages/detail/detail?item=${encodeURIComponent(JSON.stringify(item))}`
})
// URL 变成：/pages/detail/detail?item=%7B%22name%22%3A...%7D
```

**完整的流程：**

```
对象 item
    ↓ JSON.stringify
JSON 字符串 '{"name":"景点'name","price":100}'
    ↓ encodeURIComponent
URL 安全的编码字符串 '%7B%22name%22%3A...%7D'
    ↓ 拼接到 URL
/pages/detail/detail?item=%7B%22name%22%3A...%7D
    ↓ onLoad 接收
options.item = '%7B%22name%22%3A...%7D'
    ↓ decodeURIComponent
'{"name":"景点'name","price":100}'
    ↓ JSON.parse
{ name: "景点'name", price: 100 }  ← 还原为原始对象
```

**简单记：JSON 是对象和字符串之间互相转换的桥梁。** 对象要进 URL，先 JSON.stringify；收到后 JSON.parse。

**三句话总结：**
- 为什么需要 JSON → URL 只能是字符串，对象要先转 JSON 字符串
- 四步流程图 → 对象→JSON→编码→解码→还原
- 简单记忆法 → JSON 是对象和字符串之间互相转换的桥梁

## 项目示例

### 首页 → 详情页（navigateTo）

```javascript
// index.vue
const goDetail = (item) => {
  const can = JSON.stringify(item)
  uni.navigateTo({
    url: `/pages/detail/detail?item=${encodeURIComponent(can)}`
  })
}

// detail.vue
onLoad((options) => {
  if (options.item) {
    const item = JSON.parse(decodeURIComponent(options.item))
  }
})
```

### 详情页 → 路线页（navigateTo）

```javascript
// detail.vue
uni.navigateTo({
  url: `/pages/line/line?id=${item.id}`
})
```

## 跳转方式选择

| 场景 | 推荐方式 |
|------|----------|
| 跳转普通页面，需返回 | `navigateTo` ✅ |
| 跳转普通页面，不需返回 | `redirectTo` ✅ |
| 跳转 tabBar 页面 | `switchTab` ✅（必须） |
| 重新加载整个应用 | `reLaunch`（慎用） |
| 返回上一页 | `navigateBack` ✅ |
| 模板中跳转 | `<navigator>` |

> **注意：如果跳转到 tabBar 页面请用 `switchTab`，否则用 `navigateTo` / `redirectTo`。**
