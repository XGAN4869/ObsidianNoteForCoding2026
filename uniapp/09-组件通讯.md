# 组件通讯

---

## onLoad 使用场景

`onLoad` 是 uni-app **页面**生命周期，**不是组件生命周期**。写在 pages/ 下的页面才生效。

### onLoad 什么时候用？

| 场景 | 说明 | 示例 |
|------|------|------|
| 页面加载时请求数据 | 页面一进来就需要的数据 | 请求列表、详情 |
| 接收跳转参数 | 从其他页面跳转过来时获取参数 | `options.id` |
| 初始化页面状态 | 设置页面初始值 | 设置默认选中项 |

### 简单理解

```
onLoad = 页面创建时执行 = 相当于 Vue 的 beforeRouteEnter + onMounted
```

### 项目实际例子

```javascript
// detail.vue - 加载时获取详情
onLoad((options) => {
  loadDetail(options.id)
  if (options.item) {
    const item = JSON.parse(decodeURIComponent(options.item))
  }
})

// like.vue - 加载时获取列表
onLoad(() => {
  loadLikeList()
})

// line.vue - 加载时获取路线
onLoad((props) => {
  if (props.id) fetchProjectDetail(props.id)
})
```

### onLoad vs onShow 区别

| | onLoad | onShow |
|--|--------|--------|
| 执行时机 | 页面**创建**时 | 页面**显示**时 |
| 从 B 返回 A 时 | ❌ 不执行（A 没销毁） | ✅ 执行 |
| 刷新页面 | ✅ 执行 | ✅ 执行 |
| 适合做什么 | 初始化请求、接参数 | 刷新数据、曝光统计 |

> **注意：组件（components/ 下）用 `onMounted`，不用 `onLoad`！**

## uni-app 页面生命周期 vs Vue 路由守卫

虽然功能上有部分重叠，但**本质不一样**。

### 类比表

| uni-app 页面生命周期 | Vue 路由守卫 | 说明 |
|---------------------|-------------|------|
| `onLoad` | `beforeRouteEnter` + `onMounted` | 页面进入，可接参数 |
| `onShow` | **无等价** | 页面显示（多端特有） |
| `onReady` | `nextTick` | DOM 渲染完成 |
| `onHide` | `beforeRouteLeave` | 页面离开/隐藏 |
| `onUnload` | **无等价** | 页面销毁（小程序特有） |

### 核心区别

| | Vue 路由守卫 | uni-app 页面生命周期 |
|--|------------|---------------------|
| **设计目的** | 导航控制（能不能跳转、跳转前做啥） | 页面状态管理（创建/显示/隐藏/销毁） |
| **关注点** | 路由变化 | 页面实例生命周期 |
| `onShow` / `onHide` | 无 | uni-app 有，因为多端有"切后台"概念 |
| `onUnload` | 无 | uni-app 有，因为小程序有真实页面实例销毁 |

### 简单理解

**Vue 路由守卫 = 路由层面控制（能不能跳、跳去哪）**

**uni-app 页面生命周期 = 页面层面控制（页面创建/显示/隐藏/销毁做什么）**

### 为什么 uni-app 有 onShow/onHide，Vue 没有？

因为两者架构不同：

| | Vue | uni-app |
|--|-----|---------|
| 架构 | SPA，整个项目一个实例 | 多页面，每个页面是独立实例 |
| 切换页面 | 组件切换，不创建新实例 | 创建/销毁页面实例 |
| 切到后台 | SPA 不存在这个概念 | 小程序/App 切后台，页面隐藏 |
| 切回前台 | SPA 不存在这个概念 | 页面重新显示，触发 onShow |

**uni-app 的 onShow/onHide 是多页面架构的产物，Vue SPA 没有这个概念。**

## 通讯方式总览

uni-app 中有多种通讯场景和方式：

| 通讯场景 | 方式 | 本项目是否用到 |
|----------|------|--------------|
| 父组件 → 子组件（props） | `:list="bannerList"` | ✅ 用到 |
| 子组件 → 父组件（事件） | `@click="handleClick"` | ✅ 用到 |
| 双向绑定 | `v-model` | ✅ 用到 |
| 插槽传值 | `v-slot:xxx="{item}"` | ✅ 用到 |
| 页面 → 页面（URL参数） | `navigateTo + onLoad` | ✅ 用到 |
| 页面 → 页面（本地存储） | `setStorageSync / getStorageSync` | ✅ 用到 |
| 页面 → 页面（eventChannel） | `getOpenerEventChannel` | 补充 |
| 跨页面全局通讯 | `uni.$emit / $on / $off` | 补充 |

## 一、父子组件通讯（Props / Emit / v-model）

### 1. Props 父→子传值（本项目用到）

父组件通过属性传递给子组件：

```vue
<!-- 父组件 index.vue -->
<up-swiper
  :list="bannerList"        <!-- 传递数组 -->
  :autoplay="true"          <!-- 传递布尔 -->
  radius="8"                 <!-- 传递字符串 -->
></up-swiper>
```

子组件内部接收（uview-plus 已封装好，外部使用不需要管）：

```javascript
// 子组件内部类似这样接收
props: {
  list: Array,
  autoplay: Boolean,
  radius: {
    type: String,
    default: '0'
  }
}
```

### 2. Emit 子→父事件（本项目用到）

子组件触发事件，父组件监听：

```vue
<!-- 父组件 my.vue -->
<up-popup :show="show" @close="close">
  ...
</up-popup>

<!-- up-popup 内部触发 close 事件 -->
<!-- 父组件监听 @close="close" 即可 -->
```

```vue
<!-- 父组件 detail.vue -->
<view class="item" @click="goLine(item)">
  <!-- 子组件触发点击事件，父组件处理 -->
</view>
```

### 3. v-model 双向绑定（本项目用到）

```vue
<!-- index.vue -->
<up-search
  v-model="keyword"           <!-- 双向绑定搜索词 -->
></up-search>

<!-- up-waterfall -->
<up-waterfall v-model="flowList">
  <!-- v-model 实现双向数据绑定 -->
</up-waterfall>
```

**v-model 本质**：`:modelValue` + `@update:modelValue`

```vue
<!-- 上面等价于 -->
<up-search
  :modelValue="keyword"
  @update:modelValue="keyword = $event"
></up-search>
```

### 4. 插槽 v-slot 父→子传值（本项目用到）

```vue
<!-- 父组件 index.vue -->
<up-waterfall v-model="flowList">
  <template v-slot:left="{leftList}">
    <!-- leftList 是子组件传过来的数据 -->
    <view v-for="item in leftList" :key="item.id">
      {{ item.title }}
    </view>
  </template>
</up-waterfall>
```

## 二、页面间通讯

### 1. URL 参数传递（本项目用到）

适合传递简单参数：

```javascript
// A 页面跳转 B
uni.navigateTo({
  url: '/pages/detail/detail?item=' + encodeURIComponent(JSON.stringify(item))
})

// B 页面 onLoad 中接收
onLoad((options) => {
  if (options.item) {
    const item = JSON.parse(decodeURIComponent(options.item))
  }
})
```

### 2. 本地存储（本项目用到）

适合需要持久化的数据：

```javascript
// my.vue - 保存用户信息
uni.setStorageSync('userInfo', JSON.stringify(userInfo))

// my.vue - 读取用户信息
onLoad(() => {
  const cache = uni.getStorageSync('userInfo')
  if (cache) {
    Object.assign(userInfo, JSON.parse(cache))
  }
})
```

### 3. eventChannel（补充）

适合 A→B 跳转，B 处理完后返回 A 并传参：

```javascript
// A 页面
const handleClick = () => {
  uni.navigateTo({
    url: '/pages/list/list',
    events: {
      acceptData: (data) => {
        console.log('收到 B 的数据：', data)
      }
    }
  })
}

// B 页面
uni.navigateBack({
  success: () => {
    const eventChannel = getOpenerEventChannel()
    eventChannel.emit('acceptData', { name: '来自 B' })
  }
})
```

## 三、全局跨页面通讯（uni.$emit）

适合跨多个页面的事件通知：

### 原理

```
uni.$emit('eventName', data)  →  所有监听 'eventName' 的页面都能收到
```

### 使用场景

适合：推送通知、登录状态更新、购物车数量变化等跨页面同步。

### 示例

```javascript
// A 页面 - 监听
onLoad(() => {
  uni.$on('updateCart', (count) => {
    console.log('购物车更新了：', count)
  })
})

onUnload(() => {
  uni.$off('updateCart')  // 务必取消监听
})

// B/C/任意页面 - 触发
uni.$emit('updateCart', 5)
```

### 生命周期要点

| 操作 | 写在哪 | 为什么 |
|------|--------|--------|
| `uni.$on` 绑定 | `onLoad` | 页面创建时绑定，只绑一次 |
| `uni.$off` 解绑 | `onUnload` | 页面销毁时清理，防止内存泄漏和重复触发 |

**如果写在 onShow 会怎样？** 每次 `onShow` 都会绑定新监听，导致同一个事件触发多次。

## 四、本项目通讯实际案例

### 1. 首页 → 详情页（URL参数 + 事件）

```javascript
// index.vue - 跳转并传参
const goDetail = (item) => {
  const can = JSON.stringify(item)
  uni.navigateTo({
    url: `/pages/detail/detail?item=${encodeURIComponent(can)}`
  })
}

// detail.vue - onLoad 接收
onLoad((opt) => {
  if (opt && opt.item) {
    const data = JSON.parse(decodeURIComponent(opt.item))
    Object.assign(itemData, data)
  }
})

// detail.vue - 点击跳转路线页
const goLine = (item) => {
  uni.navigateTo({
    url: `/pages/line/line?id=${item.id}`
  })
}
```

### 2. my.vue 用户信息存储（本地存储）

```javascript
// 保存
uni.setStorageSync('userInfo', JSON.stringify(userInfo))

// 读取
onLoad(() => {
  const cache = uni.getStorageSync('userInfo')
  if (cache) Object.assign(userInfo, JSON.parse(cache))
})
```

### 3. 组件内父子通讯

```vue
<!-- 父组件 -->
<up-popup :show="show" @close="close">
  <input v-model="userInfo.nickName">
  <button @click="userSubmit">提交</button>
</up-popup>

<!-- up-popup 内部触发 -->
<!-- @close="close" 监听关闭事件 -->
<!-- :show="show" props 控制显示 -->
```

## 五、通讯方式选择指南

| 场景 | 推荐方式 |
|------|----------|
| 父组件给子组件传值 | props |
| 子组件通知父组件 | emit / @事件名 |
| 双向绑定 | v-model |
| 插槽获取子组件数据 | v-slot |
| 页面跳转传简单参数 | URL 参数 + onLoad |
| 页面跳转返回参数 | eventChannel |
| 需要持久化的数据 | StorageSync |
| 跨多个页面的事件 | uni.$emit / $on |

## 七、补充：Pinia / Vuex 状态管理

当项目页面多、数据共享场景复杂时，简单的 `uni.$emit` 或 `StorageSync` 不够用，需要状态管理库。

### Pinia（Vue 3 主流，uni-app 推荐）

**特点：**
- 比 Vuex 更轻量、更简单
- 支持 `defineStore` 定义 store
- 自动追踪变化，响应式

**项目结构：**

```
store/
├── user.js       ← 用户状态
└── cart.js       ← 购物车状态
```

**示例：**

```javascript
// store/user.js
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    nickName: '',
    avatarUrl: '',
    token: ''
  }),
  actions: {
    setUser(userData) {
      this.nickName = userData.nickName
      this.avatarUrl = userData.avatarUrl
    },
    clearUser() {
      this.nickName = ''
      this.avatarUrl = ''
      this.token = ''
    }
  }
})

// 任意页面使用
import { useUserStore } from '@/store/user.js'

const userStore = useUserStore()

// 读取
console.log(userStore.nickName)

// 修改
userStore.setUser({ nickName: '张三', avatarUrl: 'xxx' })

// 清除
userStore.clearUser()
```

### Pinia vs uni.$emit 对比

| 场景 | 用 Pinia | 用 uni.$emit |
|------|----------|-------------|
| 用户登录状态 | ✅ 适合，多页面共享 | ⚠️ 能用但麻烦 |
| 购物车数量 | ✅ 适合，实时响应 | ⚠️ 需要手动同步 |
| 简单页面跳转传参 | ❌ 杀鸡用牛刀 | ✅ 适合 |
| 跨多个页面的持久状态 | ✅ 适合 | ❌ 不适合 |

### 什么时候用 Pinia？

- 项目页面多（5个以上）
- 多个页面需要共享同一份数据
- 数据需要持久化（配合 Storage）
- 不想一层层 props 传值

### 什么时候不用 Pinia？

- 项目小（3-4个页面）
- 页面间传参简单（URL 参数够用）
- 数据不需要共享（各自页面管各自的）

## 八、老师笔记整理

### 页面跳转传值

```javascript
// A 页面：跳转并传参
uni.navigateTo({
  url: `/pages/search/search?key=${this.key}&id=111`
})

// B 页面：onLoad 中接收
onLoad((options) => {
  this.keyName = options.key
  this.id = options.id
})
```

### 组件传值

```javascript
// 父传子（props）- 和 Vue 一样
// 父：
<ChildComponent :list="list" />

// 子：
props: {
  list: Array
}
```

### 全局事件传值

```javascript
// A 页面：监听
uni.$on('名称', (参数值) => {
  console.log('收到：', 参数值)
})

// B 页面：触发
uni.$emit('名称', 值)
```
