# CLAUDE.md

This repository is an Obsidian vault, not a traditional software project.

## Repository Focus

- Work mainly with Markdown notes, Obsidian links, properties, tags, embeds, and vault organization.
- Prefer note editing, restructuring, summarizing, and linking over software-style assumptions like builds or test pipelines.
- Many notes are written in Chinese.

## Important Paths

- `面试整理/` - interview and study notes
- `Blog/` - draft blog content
- `.obsidian/` - Obsidian config and plugins
- `.agents/skills/` - local Claude Code skills

## Working Style

- Keep changes focused and minimal.
- Read only the files needed for the current task.
- Preserve existing Obsidian syntax such as wikilinks, frontmatter, embeds, and callouts.
- Treat this file as durable project memory; do not rely on old chat history for long-term rules.

## Context Management

- Prefer one concrete task per session.
- If a task gets long, summarize progress briefly with:
  - goal
  - relevant files
  - current status
  - next step
- Suggest `/compact` when the thread is getting large.
- Suggest `/clear` or a fresh `cc` session when the topic has shifted or the thread is noisy.

## Useful Tools

- Use local skills in `.agents/skills/` when relevant.
- Use the Obsidian CLI skill when interacting with a running Obsidian vault from the command line.
