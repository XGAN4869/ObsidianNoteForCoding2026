# Claude Code 同步系统流程

## 系统概述

这是一个用于在 **VS Code中的Claude Code扩展** 和 **终端中的Claude Code** 之间同步笔记的系统。通过共享文件系统和同步脚本，实现两个Claude Code实例间的信息传递。

## 同步脚本详解

### 脚本位置
- `C:\Users\Administrator\Desktop\备用项目\homepage-main\Zora-HomePage-main\claude-sync.sh`

### 主要功能

```bash
# 查看同步状态
./claude-sync.sh status

# 将项目笔记推送到Obsidian
./claude-sync.sh push

# 从Obsidian拉取笔记到项目
./claude-sync.sh pull

# 双向同步
./claude-sync.sh sync
```

### 脚本逻辑
1. **状态检查**：显示两个目录的笔记数量和示例文件
2. **单向同步**：使用`rsync`或`cp`命令复制较新的文件
3. **双向同步**：执行双向的`push`和`pull`

## 使用流程

### 场景1：在VS Code中记录代码讨论
1. **在VS Code Claude Code中**：
   ```bash
   # 在项目目录创建笔记
   echo "# 组件优化讨论" > claude-notes/code/组件优化.md
   cat >> claude-notes/code/组件优化.md << 'EOF'
   
   ## 问题
   用户头像组件加载慢
   
   ## 解决方案
   ```vue
   <UserAvatar :src="avatarUrl" loading="lazy" />
   ```
   EOF
   ```

2. **运行同步**：
   ```bash
   ./claude-sync.sh push
   ```

3. **在终端Claude Code中查看**：
   ```bash
   cd "E:\Work1\githubClone\ObsidianNoteForCoding2026"
   cat claude-sync/zora-homepage/code/组件优化.md
   ```

### 场景2：在Obsidian中记录学习总结
1. **在终端Claude Code中**：
   ```bash
   cd "E:\Work1\githubClone\ObsidianNoteForCoding2026"
   cat > claude-sync/zora-homepage/learning/windi-css技巧.md << 'EOF'
   # Windi CSS 使用技巧
   
   ## 响应式设计
   ```html
   <div class="text-sm md:text-base lg:text-lg">
     自适应文字大小
   </div>
   ```
   EOF
   ```

2. **运行同步**：
   ```bash
   cd "C:\Users\Administrator\Desktop\备用项目\homepage-main\Zora-HomePage-main"
   ./claude-sync.sh pull
   ```

3. **在VS Code Claude Code中查看**：
   ```bash
   cat claude-notes/learning/windi-css技巧.md
   ```

### 场景3：跨会话传递完整讨论
1. **保存当前Claude Code会话**：
   ```bash
   # 将会话内容保存到文件
   echo "## 会话记录 $(date)" > claude-notes/sessions/$(date +%Y-%m-%d-%H-%M).md
   # 将剪贴板内容或对话历史追加到文件
   ```

2. **同步到Obsidian**：
   ```bash
   ./claude-sync.sh push
   ```

3. **在另一个Claude Code实例中继续**：
   ```bash
   # 查看之前的讨论
   cat claude-sync/zora-homepage/sessions/最新文件.md
   # 基于之前的讨论继续工作
   ```

## 文件命名约定

为了更好的组织，建议使用以下命名格式：

### 代码讨论
- `功能-问题描述.md`：`登录表单-验证优化.md`
- `组件名-优化.md`：`UserMenu-响应式优化.md`
- `bug-问题描述.md`：`bug-首页白屏.md`

### 学习总结
- `技术名-主题.md`：`vue3-组合式API.md`
- `工具名-技巧.md`：`git-分支策略.md`
- `概念-解释.md`：`SSR-原理.md`

### 会话记录
- `YYYY-MM-DD-HH-MM.md`：`2026-04-04-14-30.md`
- `YYYY-MM-DD-主题.md`：`2026-04-04-项目架构.md`

## 注意事项

1. **文件冲突**：如果两端同时修改了同一个文件，后同步的会覆盖先同步的
2. **大文件**：避免同步过大的文件（如图片、二进制文件）
3. **敏感信息**：不要在笔记中包含密码、API密钥等敏感信息
4. **备份**：定期备份重要笔记到Git或其他位置
5. **清理**：定期清理旧的或不再需要的笔记文件

## 扩展建议

### 1. Git集成
```bash
# 将claude-notes添加到.gitignore
echo "claude-notes/" >> .gitignore
# 或单独提交笔记
git add claude-notes/
git commit -m "docs: 更新开发笔记"
```

### 2. 定时同步
```bash
# 使用cron定时同步（Linux/Mac）
# 每30分钟同步一次
*/30 * * * * cd /path/to/project && ./claude-sync.sh sync

# Windows使用任务计划程序
```

### 3. VS Code任务集成
在`.vscode/tasks.json`中添加：
```json
{
  "label": "同步Claude笔记",
  "type": "shell",
  "command": "./claude-sync.sh sync",
  "group": "build"
}
```
